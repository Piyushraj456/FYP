  1. PROJECT OVERVIEW
================================================================================

  LegalEase is an AI-powered legal document analysis platform developed as a
  Final Year Capstone Project. The system is designed to automate the review
  of complex legal contracts and agreements, making legal analysis faster,
  more accessible, and easier to understand.

  Users can upload any PDF document — including scanned files — and the system
  will automatically extract key clauses, detect potential legal risks, generate
  plain-English summaries, and provide actionable recommendations through a
  structured interactive dashboard.

  The project integrates Natural Language Processing (NLP), Machine Learning
  (ML), and Large Language Models (LLMs) into a production-grade full-stack
  web application, demonstrating real-world applied AI in the legal technology
  domain.

  Key Features:
    > Automated clause extraction and classification
    > Risk detection using rule-based logic and ML models
    > LLM-generated plain-English contract summaries
    > Overall contract risk scoring
    > Secure user authentication and document management
    > OCR support for scanned PDF documents

================================================================================
  2. SYSTEM ARCHITECTURE
================================================================================

  LegalEase is built on a three-tier microservices architecture, with each
  service responsible for a distinct layer of functionality.

  +--------------------------------------------------------------------------+
  |  Layer              Folder        Technology Stack                        |
  +--------------------------------------------------------------------------+
  |  AI / NLP Service   zlegal/       Python 3.10, FastAPI, spaCy,           |
  |                                   pdfplumber, pytesseract (OCR)          |
  |                                                                          |
  |  Backend API        zlegalai/     Node.js, Express.js, MongoDB,          |
  |                                   JWT Authentication, UploadThing        |
  |                                                                          |
  |  Frontend UI        zzlegalai/    React.js, Vite, Axios, UploadThing     |
  +--------------------------------------------------------------------------+

  Service Communication:
    - The React Frontend communicates with the Node.js Backend via REST API.
    - The Node.js Backend forwards document processing requests to the
      Python FastAPI AI Service.
    - Processed results are stored in MongoDB and returned to the frontend.

  4. INSTALLATION & SETUP
================================================================================

  ------------------------------------------------------------------------
  4.1  CLONE THE REPOSITORY
  ------------------------------------------------------------------------

    git clone <your-repo-link>
    cd <project-folder>

  ------------------------------------------------------------------------
  4.2  PYTHON AI SERVICE  —  zlegal/
  ------------------------------------------------------------------------

    Navigate to the folder:
      cd zlegal

    Create a Python virtual environment:
      python -m venv venv

    Activate the virtual environment:
      Windows  :  venv\Scripts\activate
      Mac/Linux:  source venv/bin/activate

    Install all required dependencies:
      pip install -r requirements.txt

    Download the spaCy NLP language model:
      python -m spacy download en_core_web_sm

    Start the FastAPI server:
      uvicorn main:app --host 0.0.0.0 --port 8000 --reload

    Confirm the service is running:
      Service URL  →  http://localhost:8000
      API Docs     →  http://localhost:8000/docs

  ------------------------------------------------------------------------
  4.3  NODE.JS BACKEND  —  zlegalai/
  ------------------------------------------------------------------------

    Navigate to the folder:
      cd ../zlegalai

    Install all required dependencies:
      npm install

    Create a .env file in the root of this folder:
      touch .env   (Mac/Linux)
      OR manually create a file named ".env"

    Add the following environment variables to the .env file:

      PORT=5000
      MONGO_URI=your_mongodb_connection_string
      JWT_SECRET=your_jwt_secret_key
      UPLOADTHING_SECRET=your_uploadthing_secret
      UPLOADTHING_APP_ID=your_uploadthing_app_id
      FASTAPI_URL=http://localhost:8000

    Start the backend server:
      npm start

    Confirm the service is running:
      Backend URL  →  http://localhost:5000

  ------------------------------------------------------------------------
  4.4  REACT FRONTEND  —  zzlegalai/
  ------------------------------------------------------------------------

    Navigate to the folder:
      cd ../zzlegalai

    Install all required dependencies:
      npm install

    Create a .env file in the root of this folder:
      touch .env   (Mac/Linux)
      OR manually create a file named ".env"

    Add the following environment variables to the .env file:

      VITE_API_URL=http://localhost:5000
      VITE_UPLOADTHING_APP_ID=your_uploadthing_app_id

    Start the frontend development server:
      npm run dev

    Confirm the application is running:
      Frontend URL  →  http://localhost:5173

================================================================================

  Supported File Formats:
    - Standard PDF files          (.pdf)
    - Scanned / Image-based PDFs  (.pdf with OCR processing)

  Document Requirements:
    - English language preferred for optimal NLP accuracy
    - Clear, legible text for best OCR results on scanned files
    - No password protection on the PDF file

================================================================================
  8. PROCESSING PIPELINE
================================================================================

  Uploaded documents pass through a three-stage processing pipeline:

  STAGE 1  |  EXTRACTION
  -----------------------------------------------------------------------
  > Text is extracted from standard PDFs using pdfplumber.
  > Scanned or image-based PDFs are processed through OCR (pytesseract)
    to recover the full text content.

  STAGE 2  |  ANALYSIS
  -----------------------------------------------------------------------
  > Extracted text is segmented into individual clauses.
  > Each clause is classified by type (e.g., liability, indemnity,
    termination, confidentiality, payment terms, etc.).
  > Risk detection is applied using a combination of rule-based logic
    and trained ML classification models.

  STAGE 3  |  REPORT GENERATION
  -----------------------------------------------------------------------
  > An LLM generates a plain-English summary of the full document.
  > An overall risk score is calculated for the contract.
  > Specific recommendations are generated for flagged clauses.
  > Results are stored in MongoDB and displayed on the dashboard.

================================================================================
  9. OUTPUT & REPORT FORMAT
================================================================================

  Each analyzed document produces a structured report containing:

  (A) CLAUSE ANALYSIS
      - Clause type and classification label
      - Risk flags with detailed explanations
      - Contextual notes for each identified clause

  (B) RISK SCORE
      - An overall contract risk rating (Low / Medium / High)
      - Breakdown of risk by clause category

  (C) PLAIN-ENGLISH SUMMARY
      - A concise, human-readable summary of the full document
      - Written to be understandable without legal expertise

  (D) RECOMMENDATIONS
      - Specific, actionable suggestions for improving high-risk clauses
      - Prioritized by severity of detected risk

  Output is:
    - Displayed interactively on the frontend dashboard
    - Persisted in the MongoDB database for future reference

    http://localhost:8000/docs  (Swagger UI — auto-generated by FastAPI)

================================================================================
  11. TROUBLESHOOTING GUIDE
================================================================================

  Issue    :  FastAPI service not responding
  Solution :  Ensure the Python virtual environment is activated and the
              uvicorn server is running on port 8000. Check terminal for
              error messages.

  Issue    :  Node.js backend throws an error on startup
  Solution :  Verify that the .env file exists in zlegalai/ and that all
              required environment variables are correctly filled in.
              Confirm MongoDB is running and the connection string is valid.

  Issue    :  Frontend cannot connect to the backend
  Solution :  Check that VITE_API_URL in zzlegalai/.env is set to
              http://localhost:5000 and that the Node.js backend is running.

  Issue    :  File upload fails
  Solution :  Verify that UPLOADTHING_SECRET and UPLOADTHING_APP_ID are
              correctly configured in both zlegalai/.env and zzlegalai/.env.
              Confirm your UploadThing account is active.

  Issue    :  OCR not working on scanned PDFs
  Solution :  Ensure pytesseract and Tesseract OCR are properly installed
              on your system. Refer to the requirements.txt for exact
              versions.
